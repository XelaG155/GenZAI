Was ist eine VM?

Virtuelle Maschine = Computer-in-the-Cloud. Du mietet CPU/GPU, installierst dein Stack und shipst. Vorteil: volle Kontrolle, Nachteil: du managest Patches & Kosten.

  • CPU-VM: g√ºnstig, gut f√ºr API/RAG ohne heavy Modelle.
  • GPU-VM: wenn du lokal inferencen willst (Llama 3, Stable Diffusion).
  • Serverless: Functions f√ºr schlanke Tasks, kein eigener Server.

Setup in 20 Minuten

  1. Provider w√§hlen (AWS, GCP, Azure oder Hetzner/Scaleway f√ºr Budget).
  2. Ubuntu-VM starten (>= 4 vCPU, 16GB RAM f√ºr RAG; GPU: A10/A100/4090 je nach Budget).
  3. SSH + Updates: sudo apt update && sudo apt upgrade -y.
  4. Docker installieren: curl -fsSL https://get.docker.com | sh; sudo usermod -aG docker $USER.
  5. Compose your stack: API (FastAPI), Vector DB (Chroma), Ingress (NGINX/Traefik).
  6. Secrets via .env / Cloud Secret Manager, nie im Repo.
  7. Monitoring quick: docker logs, Healthchecks, UptimeRobot.

Install AI Tooling

  • Python 3.11 + venv: python3 -m venv .venv && source .venv/bin/activate.
  • Core libs: pip install fastapi uvicorn openai anthropic langchain chromadb tiktoken.
  • GPU? CUDA + pip install torch --index-url https://download.pytorch.org/whl/cu121.
  • Model Hosting: Ollama, vLLM, TGI oder Hugging Face Inference Endpoints.

Security & Costs

  • SSH Keys only, Security Groups nur 80/443/22.
  • API Keys in Secrets, nicht in Frontend bundeln.
  • Autosleep f√ºr Dev-VMs, Budgets setzen.
  • Logs anonymisieren, PII minimieren.
